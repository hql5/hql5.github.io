第二章：进程描述和控制

进程状态

  • 进程总是按照一定的顺序执行指令集，通过列出指令集来描述单个进程的方法叫做进程的跟踪。
  • 三种基本状态:★
    • 运行：占有CPU,正在CPU上执行
    • 就绪：进程已获得除CPU之外的所有必要资源，一旦有CPU就可运行。
    • 阻塞：因某事件暂停执行，例如等待I/O操作，申请缓冲空间等。 通常将阻塞的进程排成一个队列，成为阻塞队列。
    • 创建
    • 终止

进程挂起★

  • 原因：在内存中所有进程都等待I/O设备的现象是很普遍的，即使同时运行多个程序，CPU大多数之间仍旧是空闲状态。
原因 说明
交换 OS需要释放足够的内存空间调入就绪进程执行
其他OS原因 OS挂起一个后台进程或怀疑引起问题的进程
交互用户请求 用户要求挂起
间断 周期性执行的一个进程，在等待下次执行时
父进程的请求 父进程协调多个子进程的运行
  • 解决办法：
    • 增加主存（费用增加）；
    • 引入交换技术：（实质上是一个I/O操作）当内存中没有进程处于就绪时，OS将其中一个阻塞进程转移到磁盘的挂起队列中。使CPU和内存可以处理响应新的进程。
    • 阻塞—（挂起）–>挂起—（激活）–>就绪。

进程描述

操作系统控制结构（四类表管理信息）

  • 内存表（memory table）：跟踪主存和辅存。
  • I/O表：管理I/O设备和通道。
  • 文件表：提供当前存在文件中的信息（文件位置、状态、其他性质）
  • 进程表：管理进程。

进程控制属性

  • PCB进程控制块★ PCB是进程实体的一部分，是OS中最重要的记录性数据结构，它记录了操作系统所需的、用于描述进程情况及控制进程运行所需的全部信息。
基本元素 包含内容
进程标识 本进程ID；父进程、父子关系PID/PPID；用户标识
CPU状态信息 各种寄存器（PC，状态码，状态信息）；栈指针（每个进程都有一个LIFO的系统栈）
进程状态信息 当前状态；进程优先级；调度相关信息，开始等待事件
进程控制信息 计时信息；进程间通信信息，特权信息
进程资源信息 进程拥有的资源（CPU，内存，设备，文件）
数据结构 可能会包括指向其他进程（同队列、砼结构）的指针

进程控制

系统中通过一些特定功能的程序段（原语）来完成进程的创建、终止以及各种状态的切换。

执行模式★

  • 内核模式（系统态（管态））：可以执行全部指令，调用全部资源
    • 主要功能：进程管理，设备管理，存储器管理；中断处理，时钟管理，源于操作
    • 原语：由若干个机器指令构成的完成某种特定功能的程序段，具有不可分割性，执行过程中不允许被中断。（不可中断性（原子性））
  • 用户模式（用户态）：只能执行用户指令

进程创建过程★

  1. 给新进程一个编号，并申请PCB。此时，在进程表中增加新条目。
  2. 为进程分配空间。
  3. 初始化PCB。除了PC和系统栈指针外，其他状态信息部分大都初始化为0。状态通常设置为就绪或就绪挂起。优先级一般设置为最低级。除了明确要求或从父继承外，一般没有I/O、文件资源。
  4. 设置合适的链接。常见的PCB管理组织形式有：链式，索引式，基本的组织形式是数组。
  5. 生成一些其他数据结构。

线程★

  • 引入：进程的引入是为了程序并发执行，改善资源利用率及提高系统的吞吐量。为了是进程并发执行，系统必须进行额外的开销。如何提高程序的并发程度同时又减少操作系统的开销？因此引入线程。

  • 优点：进一步提高了程序并发度，提高了系统的吞吐量和并发粒度。

  • 定义★：线程是进程内的一个相对独立的、可独立调度和指派的执行单元。

  • 特征（性质）：

    1. 线程是进程内的一个相对独立的可执行单元。
    2. 是OS中的基本调度单元，线程中包含调度所需的信息。
    3. 一个进程至少有一个线程，也可以有多个线程，因为进程不再是被调度的单元。
    4. 线程并不拥有资源。资源分配的单位是进程。
    5. 线程在需要时可以创建其他线程。

  • 线程与进程的区别和联系★

    1. 调度。线程时调度的基本单位，进程时资源分配的基本单位。
    2. 拥有资源。线程不拥有系统资源，但是可以访问隶属进程的系统资源。
    3. 并发性。不仅进程间可以并发，同一个进程中多线程也可以并发。并发程度提升。
    4. 系统开销。减小。因为线程基本不占用系统资源。

  • 线程的状态和管理：

    • 线程不具有挂起状态。因为线程部管理资源，挂起是将资源由内存转到外存。
    • 对于多线程的进程。一个线程阻塞不一定导致整个进程阻塞。
    • 进程不再是调度单位。因此进程分为：活动和非活动状态。
    • 线程使用TCB（Thread Control Block）来描述数据结构

多处理机SMP

按照通信方式划分两类：

  1. 共享存储器多处理系统：多个处理器共享一个共享存储器
  2. 分布式存储器系统（集群）

特点：

  1. 系统中有多个处理器
  2. 所有处理器共享主存储器。每个处理器有自己的私有cache–>“cache一致性问题”
